directives - Директивы — это специальные атрибуты с префиксом v-
v-model         |  используется там, где нужно динамически поменять текст
v-bind          |  добавить атрибут элементам, сокращенно :class
v-for           |  перебирает массивы и обьекты
v-show          |  показывает, скрывает и элемент 
v-if, v-else    |  показывает по-очереди тот или другой тег
v-if            |  убирает элемент, если он не соответствует условию
v-cli           |  $ vue init [template_name](webpack, webpack-simple, browserify, browserify-simple, pwa, simple) [my-project] -- (readme) npm run dev
API Global param
el:             | Привязывает экземпляр vue к определенному элементу(идентификатору, классу, тегу - body...) 
data:           |  Object, Function _ Прежде всего, это данные  — не стоит передавать сюда объекты с собственным управлением внутренним состоянием.
props:          | Array_string, Object _ Список или хэш атрибутов, по которым разрешено получение данных от родительского компонента. 
                  Передайте простой массив названий или объект, задающий дополнительные параметры, например типы, правила валидации и значения по умолчанию. 
computed:       | Вычисляемые свойства, которые будут подмешаны к экземпляру Vue. В геттерах и сеттерах this будет указывать на экземпляр Vue.
methods:         | Методы, которые будут подмешаны к экземпляру Vue. Вы можете запустить эти методы прямо из экземпляра VM, или использовать их в директивах. 
template:        | string для шаблонов. Можно вставлять как целиком html разметку так и ссылаться на готорые класси или ид
model:           | boolean(скорее всего используется для чек-боксов и радио-кнопок
